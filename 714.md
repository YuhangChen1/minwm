# 2026-07-14 对话与实验工作总结

本文记录本轮对话中围绕 Full-Duplex Action2V 微调所做的代码改造、实验、结论和未完成事项，供后续继续实验时直接接手。

## 1. 本轮目标

最初的核心怀疑是：此前使用的 `Wan21/Action2V/ar_diffusion_tf` checkpoint 可能过于“顽固”，导致新增任务模块难以微调。因此，本轮将基础 checkpoint 切换为：

```text
ckpts/Wan21/Action2V/bidirectional/model.pt
```

并继续保留、比较两种训练方案：

1. `Autoregressive rollout + 跨轮 BPTT`
2. `Previous-GT -> Next-GT teacher forcing + 逐轮 backward`

原定工作顺序为：

0. 更换为 bidirectional checkpoint；
1. 空间 token stride 从 8 降到 4；
2. 在 stride 4 下探测 4、8、16、30 blocks 的显存，并使用 partial activation checkpoint；
3. 增加可训练容量；
4. 同时报 teacher-forced reconstruction 和 autonomous rollout；
5. 分析和校正预测 latent 幅度，以及 world prior、Flow/state loss 权重。

## 2. 对此前实验的审阅结论

在切换 checkpoint 前，先完整审阅了 `prompt.md` 中已有实验和消融结果。

此前 `ar_diffusion_tf` checkpoint 的代表性结果为：

| 旧实验             | 设置                           | latent MSE |   cosine | predicted std | target std |
| ------------------ | ------------------------------ | ---------: | -------: | ------------: | ---------: |
| Autonomous rollout | stride 8、1 block、200 steps   |   1.266472 | 0.413235 |      1.170548 |   0.828415 |
| Teacher forcing    | stride 8、30 blocks、100 steps |   1.266015 | 0.416657 |      1.174650 |   0.828415 |

共同问题：

- 两者都明显差于 zero-latent baseline MSE `0.696027`；
- predicted latent std 约为 `1.17`，比 target `0.828` 高约 41%；
- 解码 RGB 仍是明显彩色噪点；
- 旧的 AR 与 teacher-forcing 结果并不是严格对照：checkpoint 初始化、blocks 数和训练步数均不同；
- 旧 teacher-forcing 还依赖 AR warm-start，无法区分训练协议与初始化的影响；
- 因此，旧结果不能单独证明是 teacher forcing、AR 协议或 checkpoint 本身造成失败。

## 3. 本轮完成的代码改造

本轮修改了 9 个现有文件并新增 1 个配置文件，当前 diff 约为 `486 insertions / 38 deletions`。改动尚未提交 Git。

### 3.1 独立的 bidirectional 实验配置

新增：

```text
full_duplex/configs/overfit_bidirectional.yaml
```

它继承已有 `overfit.yaml`，仅覆盖：

- `base_checkpoint`
- `cache_path`
- `output_dir`

同时在 `full_duplex/config.py` 中加入 YAML `extends`、递归 mapping merge 和循环继承检测。这样 bidirectional 的缓存、checkpoint 和结果不会污染旧的 `ar_diffusion_tf` 实验。

### 3.2 两种训练方案的公平初始化与可控 activation checkpoint

修改了：

- `full_duplex/train_teacher_forcing.py`
- `full_duplex/train_overfit.py`
- `full_duplex/control_training.py`
- `full_duplex/training.py`

主要能力包括：

- teacher forcing 可通过 `--config` 从相同随机种子全新初始化，不再强制从 AR checkpoint warm-start；
- AR 与 teacher forcing 都支持 `--checkpoint-blocks N`；
- 可以只对执行的前 N 个 backbone blocks 做 activation checkpoint，因而能够测试 partial checkpoint；
- AR 可控制跨轮 BPTT，teacher forcing 保持 previous-GT 输入和逐轮 backward；
- 修复推理阶段仍执行 step-0 gradient probe 的问题：该探针现在只在 grad enabled 时运行。

### 3.3 增加可训练容量

修改了：

- `full_duplex/model.py`
- `full_duplex/training.py`
- 两个训练 CLI 和 controller

新增参数：

```text
--train-last-backbone-blocks N
--backbone-learning-rate-multiplier X
```

实现内容：

- 可选择性解冻“实际执行的”最后 N 个 backbone blocks；
- 可单独解冻 pretrained base world output head；
- optimizer 为这些 backbone 参数建立独立 parameter group 和学习率倍率；
- run manifest、metrics、checkpoint 和 reload 路径都记录这部分容量；
- 对全 backbone、LoRA 和 selective block 解冻的互斥组合做了校验。

### 3.4 双协议预测和评估

修改了：

- `full_duplex/predict_checkpoint.py`
- `full_duplex/evaluate_predictions.py`

预测现在支持：

```text
--evaluation-mode trained
--evaluation-mode teacher_forced
--evaluation-mode autonomous
```

同一个 checkpoint 可以在不加载 optimizer 的情况下，严格切换 world-input 协议。预测 manifest 会记录：

- 每一轮 world input 来自 GT 还是模型上一轮预测；
- 是否使用了 GT；
- 实际 evaluation mode；
- checkpoint 和训练协议来源。

评估新增：

- predicted/target 全局 std 和比例；
- 每通道 mean、std 和 std ratio；
- 仅匹配全局 mean/std 后的 MSE；
- 最优全局 affine scale、bias 和 affine-calibrated MSE；
- checkpoint 中各任务模块的参数范数；
- 训练容量和 Flow/state/camera loss 权重记录。

这避免了把 teacher-forced reconstruction 错报成 autonomous prediction。

### 3.5 紧凑 checkpoint

由于实验中触发了用户目录磁盘配额限制，加入：

```text
--compact-checkpoint
```

紧凑模式：

- 不保存 optimizer 和 scheduler state；
- 只保存一个实际 `best.pt`；
- `latest.pt` 使用链接避免重复占用空间；
- 可以用于 reload 验证和推理，但不能无损恢复 optimizer 后继续训练。

容量增强实验使用了这一模式。为释放配额，本轮部分重复的 step checkpoint 已删除，但各有效实验的 `best.pt`、metrics、manifest、evaluation 和预测文件仍保留。

## 5. Stride 4 下的显存探测

### 5.1 Teacher forcing + 逐轮 backward

所有设置均成功跑完至少 1 个训练 step：

| 执行 blocks | checkpoint blocks | 未 checkpoint blocks |   峰值显存 | step 1 用时 | step 1 state MSE | 状态 |
| ----------: | ----------------: | -------------------: | ---------: | ----------: | ---------------: | ---- |
|           4 |                 4 |                    0 |  9.764 GiB |     43.25 s |         1.684187 | 成功 |
|           8 |                 4 |                    4 | 45.602 GiB |     37.72 s |         1.897584 | 成功 |
|          16 |                12 |                    4 | 48.450 GiB |     64.75 s |         2.324567 | 成功 |
|          30 |                26 |                    4 | 53.458 GiB |    107.63 s |         2.997883 | 成功 |

结论：在 80 GiB GPU 上，teacher forcing 的 stride 4 可以执行全部 30 blocks，并保留最后 4 个 block 不做 activation checkpoint。

注意：不同 block 数的 step-1 MSE 上升不能解释为“更多 block 更差”，因为这是不同 hidden distribution 经过新初始化 task head 的起点，而不是收敛后的公平比较。

### 5.2 Autoregressive rollout + 跨轮 BPTT

| 执行 blocks | checkpoint blocks | 峰值显存/失败点 | step 1 用时 | step 1 state MSE | 状态 |
| ----------: | ----------------: | --------------: | ----------: | ---------------: | ---- |
|           4 |                 4 |      35.830 GiB |     30.74 s |         2.085606 | 成功 |
|           8 |                 7 |    约 79.16 GiB |           - |                - | OOM  |
|           8 |                 8 |      49.956 GiB |     46.02 s |         3.132574 | 成功 |
|          16 |                16 |    约 79.09 GiB |           - |                - | OOM  |
|          30 |                30 |    约 79.16 GiB |           - |                - | OOM  |

4-block AR 的 early-turn future gradient norm 为 `0.100607`，确认跨轮梯度确实存在。

结论：stride 4、19-turn 跨轮 BPTT 的实际上限是 8 blocks，而且 8 blocks 必须全部 activation checkpoint；16/30 blocks 即使全部 checkpoint 仍超出 80 GiB。

OOM 记录保存在对应实验目录的 `exception.log` 中。

## 6. Stride 8 -> 4 的严格对照

为了隔离 stride，重新从相同 seed、相同新初始化、相同 bidirectional checkpoint 训练，统一使用 4 blocks、19 turns、10 denoising steps、20 optimization steps。

### 6.1 训练集 state loss 与显存

| 训练方案        | stride | step 20 state loss |   峰值显存 |
| --------------- | -----: | -----------------: | ---------: |
| Teacher forcing |      8 |           1.571447 |  7.047 GiB |
| Teacher forcing |      4 |           1.553380 |  9.764 GiB |
| AR + BPTT       |      8 |           1.668077 | 15.176 GiB |
| AR + BPTT       |      4 |           1.693113 | 35.830 GiB |

相对 stride 8：

- Teacher forcing 的 stride 4 state loss 改善约 `1.15%`；
- AR 的 stride 4 state loss反而变差约 `1.50%`；
- stride 4 令 AR 显存从约 15.2 GiB 增至约 35.8 GiB。

所以，更密集的空间 token 在 20-step 短实验里没有带来一致收益，不能认为 stride 4 已解决噪声问题。

### 6.2 双协议评估

每个 checkpoint 都分别使用 teacher-forced 和 autonomous 协议重新生成预测。下表的 MSE/cosine 均来自新评估文件，而不是训练过程中的 loss：

| 训练方案        | stride | teacher-forced MSE | TF cosine | autonomous MSE | Auto cosine |
| --------------- | -----: | -----------------: | --------: | -------------: | ----------: |
| Teacher forcing |      8 |           1.569042 |  0.138872 |       1.696488 |    0.057451 |
| Teacher forcing |      4 |           1.550659 |  0.149229 |       1.963545 |   -0.003872 |
| AR + BPTT       |      8 |           1.624291 |  0.099100 |       1.664634 |    0.070489 |
| AR + BPTT       |      4 |           1.620041 |  0.101729 |       1.684320 |    0.057751 |

主要观察：

- Teacher forcing 在本协议内的 reconstruction 略好，但自主 rollout 有明显退化；
- stride-4 teacher-forcing checkpoint 的 gap 最大：`1.550659 -> 1.963545`；
- AR checkpoint 在两个输入协议之间更稳健，但绝对 MSE 仍远差于 zero baseline `0.696027`；
- 这验证了“双指标报告”的必要性：只看 teacher-forced reconstruction 会严重高估自主预测能力。

## 7. 增加可训练容量的实验

基线只训练 `8,190,055` 个 task-delta 参数。容量实验采用：

- stride 4；
- 4 个执行 blocks；
- 解冻最后 1 个执行 block；
- 解冻 pretrained base world head；
- 保留 world residual head 和 time-space prior；
- backbone LR = `2e-6`，即主 LR `2e-5` 的 0.1 倍；
- 总可训练参数 `57,093,031`；
- 其中 selective backbone block 为 `48,801,536` 参数；
- 训练 20 steps，使用 compact checkpoint。

### 7.1 训练结果

| 方案            | 8.19M 基线 state loss | 57.09M 容量 state loss | 相对改善 | 容量实验峰值显存 |
| --------------- | --------------------: | ---------------------: | -------: | ---------------: |
| Teacher forcing |              1.553380 |               1.536705 |    1.07% |       10.672 GiB |
| AR + BPTT       |              1.693113 |               1.652500 |    2.40% |       36.664 GiB |

容量增加带来了一致但幅度较小的改善，AR 的收益更明显。

### 7.2 容量 checkpoint 的双协议评估

| 训练方案                     | teacher-forced MSE | TF cosine | autonomous MSE | Auto cosine |
| ---------------------------- | -----------------: | --------: | -------------: | ----------: |
| Teacher forcing + last1/head |           1.533519 |  0.162496 |       1.853607 |    0.038258 |
| AR + last1/head              |           1.605018 |  0.116652 |       1.649952 |    0.075284 |

与各自 stride-4 基线相比：

- Teacher-forcing checkpoint 的 teacher-forced MSE 改善约 `1.10%`；
- 其 autonomous MSE 从 `1.963545` 改善到 `1.853607`，约 `5.60%`，但仍很差；
- AR checkpoint 的 autonomous MSE 从 `1.684320` 改善到 `1.649952`，约 `2.04%`；
- AR 的双协议 gap 仍明显小于 teacher forcing。

因此，增加靠近输出的可训练容量方向是正向的，但一次 20-step、last-1-block 实验不足以解决结构性失败，也还不能证明 8.19M 参数已经真正收敛到平台。

## 8. 噪声幅度与结构诊断

目标 latent 的固定统计量：

```text
target mean = 0.098777
target std  = 0.828415
zero-latent baseline MSE = 0.696027
```

新的 bidirectional 实验中，predicted std 通常约为 `1.03-1.12`，相当于 target 的 `1.245-1.353` 倍。相比旧 checkpoint 的约 `1.17` 有所下降，但仍高 25%-35%。

代表性结果：

| checkpoint / 评估协议 |      MSE | predicted std | std ratio | std-matched MSE | optimal affine scale | affine MSE |
| --------------------- | -------: | ------------: | --------: | --------------: | -------------------: | ---------: |
| TF stride 4 / TF      | 1.550659 |      1.050036 |  1.267525 |        1.184132 |             0.108298 |   0.673339 |
| TF stride 4 / Auto    | 1.963545 |      1.120150 |  1.352161 |        1.367042 |             0.002963 |   0.686259 |
| AR stride 4 / Auto    | 1.684320 |      1.033959 |  1.248117 |        1.314236 |             0.034035 |   0.685032 |
| TF capacity / TF      | 1.533519 |      1.054538 |  1.272959 |        1.164334 |             0.119167 |   0.670479 |
| AR capacity / Auto    | 1.649952 |      1.031582 |  1.245249 |        1.290243 |             0.048152 |   0.683803 |

关键解释：

- 幅度确实偏大，但不是唯一问题；
- 仅把输出 mean/std 对齐后，MSE 仍为约 `1.16-1.37`，依然差于 zero baseline；
- 最优全局 affine scale 大多非常接近 0，affine MSE 也只回到约 `0.67-0.69`；
- 这表示模型输出中与 target 对齐的有效结构很弱，简单乘一个 `0.828/1.05` 的缩放系数不能修复结果；
- teacher-forcing autonomous 模式的 optimal scale `0.002963` 尤其说明自主 rollout 几乎没有可线性利用的结构信号。

当前 checkpoint 中 world prior 参数 RMS 约 `7e-5`，数值很小。但现有 world-residual 模块范数统计包含 LayerNorm，不能仅凭总 RMS 判断 residual output head 的真实贡献。

本轮所有训练仍使用：

```text
lambda_flow = 1.0
lambda_state = 1.0
lambda_camera = 1.0
```

因此，“噪声幅度校正”目前只完成了统计和 post-hoc 诊断，尚未完成真正的训练消融：

- 没有做 world prior on/off 严格对照；
- 没有做 Flow/state loss 权重 sweep；
- 没有加入 latent moment/std loss；
- 没有训练可学习的输出尺度或做 per-channel calibration；
- 没有单独消融 base head、residual head 和 prior 的预测贡献。

## 12. 建议下一步

下一轮不宜直接把 stride 或 blocks 继续做大；现有证据表明核心瓶颈是预测结构没有与 target 对齐，而不只是输出 std 偏大。建议按以下顺序继续：

1. 在同一 bidirectional、stride-4、4-block、固定 seed 设置下，做 world prior on/off 严格消融；
2. 做最小 Flow/state 权重矩阵，例如保持其他条件不变，对比 `(flow,state)=(1,1)、(0.5,1)、(0.25,1)、(1,2)`；
3. 对每个候选都同时跑 teacher-forced 与 autonomous 评估，不能只看训练 state loss；
4. 若 prior 和权重无法显著提高 cosine、降低相对 zero baseline 的 MSE，再测试 per-channel moment loss 或 hidden-distribution-matched adapter；
5. 将目前最好的 AR capacity 配置延长训练，确认 20-step 改善是否持续，而不是立即增加 blocks；
6. Teacher forcing 可以继续探索更多 output-near blocks；AR 在 80 GiB 下不要尝试 16/30-block 全序列 BPTT，除非缩短历史、分段反传或改变并行/卸载策略；
7. 最终成功标准应至少同时满足：latent MSE 明显优于 `0.696027` zero baseline、cosine 明显提高、TF/Auto gap 可控、RGB 恢复出可辨认场景，而不是仅仅让 predicted std 接近 target。

## 13. 一句话结论

本轮已经把 bidirectional checkpoint 下的两条训练链路、stride 对照、显存边界、最小容量增强和双协议评估完整跑通；结果显示 stride 4 和多一点可训练容量只有小幅数值收益，RGB 仍是噪声，且真正的问题不只是 std 过大，而是 autonomous rollout 缺乏与 GT 对齐的结构信号。噪声成因的 world-prior 与 loss-weight 因果消融仍是下一轮最重要的未完成工作。

## 1. 一个 micro-turn 进入模型的结构

  一个当前时间片 t 包含：

  INPUT_STREAM_START
  [NULL_WORLD_STATE，仅 turn 0]
  world_input tokens
  MODALITY_END
  camera_input token
  MODALITY_END
  action token
  MODALITY_END
  noise tokens
  NOISE_END
  INPUT_STREAM_END

  OUTPUT_STREAM_START
  MASKED_WORLD tokens
  MODALITY_END
  MASKED_CAMERA token
  MODALITY_END
  OUTPUT_STREAM_END
  TIME_INDEX_t

  历史 turn 的输出位置不再是 mask，而是模型已经产生的 world/camera prediction：

  历史 turn:
      WORLD_OUTPUT  = historical predicted world state
      CAMERA_OUTPUT = historical predicted camera

  当前 turn:
      WORLD_OUTPUT  = MASKED_WORLD
      CAMERA_OUTPUT = MASKED_CAMERA

  GT world/camera 不放入 Transformer 序列，只用于计算监督 loss。
